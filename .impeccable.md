# .impeccable

Project-level design context for `vagabond-crawler`. Used by any skill in the Claude Design suite to keep output on-brand.

---

## Design Context

### Users

- **Primary:** GMs running dungeon-crawl sessions in the Vagabond RPG system on Foundry VTT v13+.
- **Secondary:** Players whose surfaces include the Crawl Strip, Token HUD buttons, Spell Cast Dialog, and the Merchant Shop when open.
- **Context:** Live play at the table (voice chat or in-person), with occasional prep. Decisions are made in seconds, not minutes — fiction momentum is the priority.
- **Job to be done:** Run a crawl without losing the fiction to tooling friction. Every click should feel like turning to the next page, not booting a system.

### Brand Personality

- **Three words: gritty, atmospheric, functional.**
- **Voice:** Practical GM-to-GM. Assumes tabletop RPG fluency (Favor/Hinder, TL, Close/Near/Far) and doesn't over-explain. Confident, concise, dry wit acceptable in tooltips.
- **Emotional goal:** The GM feels **in control and confident.** Informative density without visual chaos. The UI never takes focus away from the story.
- **Core tension to respect:** Atmosphere serves function, not the other way around. Never sacrifice legibility for mood.

### Aesthetic Direction

- **Reference corpus:** Video game RPG UIs — **Baldur's Gate 3** (hand-painted parchment + metal, strong headline typography), **Pathfinder: Wrath of the Righteous** (tight grids, atmospheric portraits), **Darkest Dungeon** (ornamental borders, worn textures, heavy vignettes).
- **Locked palette:** Dark charcoal surfaces + muted gold accent (`#c9a54a` / `--vcb-accent`) + fantasy semantic colors for phase, HP, and lighting. Already codified in `styles/vagabond-crawler.css` `:root` — treat as canonical.
- **Phase color grammar:**
  - Heroes = forest green `--vcb-heroes` family
  - GM = slate blue `--vcb-gm` family
  - Combat = blood red `--vcb-combat` family
  - These are **game grammar**, not decorative. Never recolor for aesthetic reasons; extend scales if needed.
- **HP gradient:** 5-stop (`--vcb-hp-ok` → `mid` → `low` → `critical` → `dead`). Already defined; use as canonical.
- **Typography:** Currently rides Foundry's default Signika + system fallbacks. Room to lean into intentional type hierarchy on flagship surfaces (Forge titles, Merchant banner, Session Recap tab headers).
- **Texture:** CSS is currently 100% gradient/solid. Room to add **sparingly-applied** parchment or metal textures to flagship surfaces (window title bars, Forge headers, Merchant banner). Never on HP bars, portraits, settings toggles, data cards.
- **Themes:** Dark is primary. Light (`body.theme-light`) must stay legible and complete but doesn't need to match dark-mode mood intensity.
- **Anti-references:**
  - Modern SaaS aesthetic (Linear / Notion / Vercel dashboards) — too clinical for this use case.
  - Neon / cyber / synthwave palettes — wrong genre.
  - Generic "AI-polish" look (purple gradients, emoji-forward decoration, glowing neons).
  - Flat plastic Material Design.

### Design Principles

1. **Legibility over mood.** The Crawl Strip is the busiest UI during play — players read it while roleplaying. Any atmospheric texture, stylization, or effect must pass the *"can the GM glance at it mid-combat and identify the active combatant in one second?"* test. If it slows the read, cut it.
2. **Phase semantics are sacred.** Heroes green / GM blue / Combat red is the game's color grammar. Never recolor these for visual variety; extend the scales instead.
3. **Atmosphere lives in the chrome, not the data.** Worn borders, ornamental dividers, drop shadows, accent glow — yes on window frames, headers, banners, forge title bars. No on HP bars, token portraits, settings toggles, quantity badges.
4. **Dark is the pilot, light is the co-pilot.** Dark mode is the home surface and gets the full aesthetic treatment. Light mode ships with every feature but keeps a lower mood ceiling — functional first, atmospheric second.
5. **WCAG AA + reduced motion.** Color contrast meets AA for text and UI controls. Never rely on color alone — always pair with icon, label, or text. Respect `prefers-reduced-motion` for every animation (fade, pulse, shake, flourish). Defeated skull, turn chevron, status pills all have non-color indicators.

### Token & CSS Conventions (for reference)

- **Prefix namespacing:** `vcb-` (bar), `vcs-` (strip), `vcl-` (light tracker), `vcm-` (merchant). Other subsystems extend the scheme (`vcfx-` for animation FX).
- **Custom properties:** All colors live as `--vcb-*` tokens on `:root` with `body.theme-light` overrides. **Never hardcode hex values in new CSS.**
- **Single file:** `styles/vagabond-crawler.css` (5200+ lines). Additions go in the matching subsystem's section with a clear comment banner.
- **Template surfaces:** 13 Handlebars templates in `templates/`. Each UI window is its own `.hbs`.
